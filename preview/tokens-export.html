Tokens export Source of truth dos tokens em formato JSON compatível com Style Dictionary / Tokens Studio. Exportado a cada minor release.

1 Formato de export

Tokens em formato hierárquico JSON. Cada folha tem value (o valor real) e type (categoria semântica). Estrutura compatível com Style Dictionary e Tokens Studio for Figma.

{
  "color": {
    "brand": {
      "blue":  { "value": "#1A3375", "type": "color" },
      "green": { "value": "#00FCA8", "type": "color" }
    },
    "neutral": {
      "surface": { "value": "#F7F8FB", "type": "color" },
      "light":   { "value": "#C8C8CC", "type": "color" },
      "strong":  { "value": "#3D3D3D", "type": "color" }
    },
    "state": {
      "success":     { "value": "#1F8F5C", "type": "color" },
      "success-bg":  { "value": "#E6F7EF", "type": "color" }
    }
  },
  "space": {
    "1": { "value": "4px", "type": "spacing" },
    "2": { "value": "8px", "type": "spacing" }
  },
  "radius": {
    "sm":   { "value": "8px",   "type": "borderRadius" },
    "md":   { "value": "16px",  "type": "borderRadius" },
    "lg":   { "value": "32px",  "type": "borderRadius" },
    "pill": { "value": "999px", "type": "borderRadius" }
  },
  "typography": {
    "display": { "value": "800 56px/1.1 'RNS Sanz'", "type": "typography" },
    "h1":      { "value": "700 40px/1.1 'RNS Sanz'", "type": "typography" },
    "body":    { "value": "400 16px/1.5 'RNS Sanz'", "type": "typography" }
  }
}
Baixar tokens.json

2 Como consumir

  • Importar tokens.json no projeto.
  • Gerar CSS variables via Style Dictionary build (ou TS direto).
  • Usar as variáveis como --color-brand-blue, --space-1, --radius-sm, etc.
  • Sem hex literais em código de produção — sempre via token.

3 Sincronização

  • Tokens vivem no .fig como source of truth.
  • Export manual a cada minor release (1.0, 1.1, etc.).
  • Patch releases (1.0.0 → 1.0.1) não geram novo export se não tocarem tokens.
  • Sempre versionar o arquivo: tokens-v1.0.json, tokens-v1.1.json.
